# 电商配图助手 — 部署说明

## 项目概述

本项目是一个基于 Dify Chatflow 的电商配图生成工具前端页面，支持上传商品图 + 输入文字需求，一键生成 AI 配图。

- **API 地址**：`https://whhongyi.com.cn/v1`
- **API Key**：配置在项目根目录 `.env` 文件中（不提交到 Git）

---

## 压缩包内容

```
电商配图助手.zip
└── dist/
    ├── index.html           # 入口页面
    └── assets/
        ├── index-xxx.css    # 样式文件
        └── index-xxx.js     # 逻辑脚本
```

---

## 部署方式

### 方式一：Nginx 部署（推荐）

1. 将 `dist/` 目录下的所有文件上传到服务器指定目录，例如 `/var/www/peitu/`

2. 配置 Nginx：

```nginx
server {
    listen 80;
    server_name your-domain.com;   # 替换为你的域名

    root /var/www/peitu;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 可选：开启 gzip 压缩以提升加载速度
    gzip on;
    gzip_types text/css application/javascript text/html;
}
```

3. 重启 Nginx：

```bash
sudo nginx -t && sudo nginx -s reload
```

4. 访问 `http://your-domain.com` 即可使用。

---

### 方式二：直接本地打开

由于项目是纯静态页面，但因为使用了 ES Module，**不能直接双击 index.html 打开**。需要用一个本地静态服务器：

```bash
# 方法 1：使用 npx（需要 Node.js）
cd dist
npx -y serve .

# 方法 2：使用 Python
cd dist
python3 -m http.server 8080
```

然后浏览器访问 `http://localhost:8080` 或 `http://localhost:3000`。

---

### 方式三：Windows IIS 部署

1. 将 `dist/` 文件夹内容复制到 IIS 站点根目录
2. 确保 MIME 类型中包含 `.js` → `application/javascript`
3. 绑定域名或端口后即可访问

---

## 修改配置

如果需要更换 API Key 或 API 地址，编辑 `dist/assets/index-*.js` 文件，搜索以下内容替换：

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| API 地址 | `https://whhongyi.com.cn/v1` | Dify 服务端地址 |
| API Key | 见 `.env` 文件 | Dify 应用密钥 |

> **提示**：如需频繁修改配置，建议在源码 `main.js` 中修改后重新执行 `npm run build` 打包。

## 功能说明

| 功能 | 说明 |
|------|------|
| 上传商品图 | 支持**点击选择**、**拖拽文件**、**Ctrl/Cmd+V 粘贴**三种方式 |
| 输入需求 | 文本框描述生成需求，支持 Shift+Enter 换行 |
| 生成图片 | 点击按钮调用 Dify API，右侧实时预览 |
| 下载图片 | 生成完成后点击「下载图片」按钮保存到本地 |
| 生成历史 | 左侧面板记录本次会话的历史（刷新后清空） |

---

## 注意事项

- 本项目为**纯前端静态页面**，无需后端服务，无需数据库
- API Key 直接暴露在前端代码中，请确保该 Key 的权限范围可控
- 生成历史仅保存在浏览器内存中，刷新页面后清空
- 如需 HTTPS，请在 Nginx 中配置 SSL 证书
